AstroShop-DevOps-Project-for-an-Astronomy-E-Commerce-Platform

This project simulates an online store selling astronomy products using a microservices architecture, providing hands-on exposure to real-world DevOps practices. Each service is implemented in a different language, enabling me to work with diverse technologies. The project covers containerization, cloud deployment, CI/CD automation, and monitoring, reflecting a complete end-to-end DevOps workflow. It also helped me understand how to design scalable, fault-tolerant systems, manage infrastructure with Terraform, and implement best practices for security and reliability.

Explore my code on GitHub !

Tools Used:

  • Containerization & Orchestration: Docker, Docker Compose, Kubernetes (EKS)
  • Cloud & Infrastructure: AWS (EC2, S3, DynamoDB, ALB Ingress Controller,EKS, Route53, IAM), Terraform
  • CI/CD & Automation: GitHub Actions, ArgoCD
  • Networking & Proxy: Envoy reverse proxy

Design Issues & Resolutions:

  • Monolith vs Microservices: Chose microservices for independent scaling and fault isolation.
  • Load Balancer vs Reverse Proxy: Used Envoy for traffic validation, SSL termination,Caching and routing.
  • Docker vs Kubernetes: Docker alone couldn’t handle scaling & self-healing → deployed on EKS.

Challenges, Best Practices and Lessons Learned

  • Kubernetes Networking: Configured ClusterIP, NodePort, and Ingress for stable communication.
  • CI/CD Failures: Fixed permissions, and used GitHub Secrets for secure automation.
  • Infrastructure Limits: Resized EBS volumes and followed Terraform best practices.
  • Monitoring & Debugging: Used kubectl to troubleshoot pods and deployments.
  • Followed least-privilege IAM policies.
  • Automated builds and deployments with CI/CD pipelines.
  • Secured Terraform state and sensitive credentials.
  • Implemented robust monitoring and logging for reliability.
  • Gained practical experience in DevOps workflows, Kubernetes orchestration, and cloud infrastructure management.
  • Learned to solve real-world architectural, CI/CD, and operational challenges efficiently.
  • Improved productivity by 60% through automated infrastructure provisioning using Terraform.
  • Reduced manual deployment effort by 80% using CI/CD pipelines for microservices.
  • Ensured automatic scaling and self-healing of services using Kubernetes, achieving 99.9% uptime.
