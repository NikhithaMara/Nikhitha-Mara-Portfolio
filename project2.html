EBS-Volume-Type-Auto-Conversion-using-AWS-Lambda-and-EventBridge

Ensuring organizational compliance for cost and performance is critical. This project demonstrates how AWS Lambda can automate EBS volume type conversion from gp2 to gp3, improving efficiency and enforcing policy. It helped me gain hands-on experience with serverless architectures and event-driven workflows.

This project automatically converts newly created EBS volumes to gp3 for better performance and cost-efficiency using a serverless, event-driven setup. It showcases a real-world scenario of cloud automation and policy enforcement.

Explore my code on GitHub !

Tools & Technologies Used:

  • AWS Lambda: Runs serverless code without managing infrastructure
  • Amazon EventBridge (CloudWatch Events): Triggers Lambda on volume creation
  • IAM Roles & Policies: Securely manage AWS service access
  • Boto3 (Python SDK): Interact with AWS EC2 APIs
  • CloudWatch Logs: Monitor and debug Lambda executions

Design Issues & Resolutions:

  • Event Handling: EBS lacks native triggers, so EventBridge acts as the listener to detect volume creation.
  • Security: IAM roles designed with least privilege to allow Lambda access only to necessary resources.
  • Monitoring: CloudWatch logs added for debugging and visibility of automated executions.

Challenges, Best Practices & Lessons Learned

  • Challenges: JSON parsing errors, event visibility issues, handling multiple events.
  • Resolutions: Validated events with CloudWatch logs, applied least privilege IAM, used correct JSON parsing.
  • Best Practices: Monitor logs, avoid hardcoding credentials, design scalable and secure workflows..
  • Enforces gp3 EBS volumes automatically.
  • Reduces manual effort and operational overhead by 80%.
  • Improves storage performance, scalability, and cost efficiency.
